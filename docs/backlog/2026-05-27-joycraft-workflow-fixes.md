### 2. Brief back-references from research and design skills

**Files:** `.agents/skills/joycraft-research/SKILL.md`, `.agents/skills/joycraft-design/SKILL.md`

Neither the `research` nor `design` skill updates the parent brief with back-references after completing their artifact. The `decompose` skill expects to find these references in the brief.

**Fix:** After writing the research/design document, each skill should:
1. Read the brief
2. Add (or update) a `> **Research:**` or `> **Design:**` link in the brief's header blockquote
3. Write the brief back

**Scope:** ~10 lines added to each skill markdown.

---

### 3. Research skill writes to flat `docs/research/` instead of per-feature layout

**File:** `.agents/skills/joycraft-research/SKILL.md`

The research skill writes to `docs/research/YYYY-MM-DD-feature-name.md` (flat layout) but the project has migrated to per-feature layout: `docs/features/<slug>/research.md`.

**Fix:** Write to `docs/features/<slug>/research.md` when a brief exists at `docs/features/<slug>/brief.md`. Fall back to flat layout if no feature folder exists.

**Scope:** ~5 lines added to the research skill.
