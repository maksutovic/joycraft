# Joysmith

> A framework for evolving software development from AI-assisted to AI-autonomous — with joy, not darkness.

**Joysmith** is a methodology and toolkit for upgrading software projects through Dan Shapiro's [5 Levels of Vibe Coding](https://www.danshapiro.com/blog/2026/01/the-five-levels-from-spicy-autocomplete-to-the-software-factory/), from "Spicy Autocomplete" (Level 0) to fully autonomous spec-to-software (Level 5).

The name is a deliberate counter-narrative to "dark factory." We believe autonomous software development should bring light and craftsmanship to engineering, not darkness.

## What's Inside

- **Research** — Deep synthesis of StrongDM's Software Factory, METR productivity study, Boris Cherny's Claude Code patterns, and more
- **Templates** — Battle-tested spec templates, implementation plan templates, CLAUDE.md boundary frameworks, and Claude Code skills
- **Assessments** — How to evaluate where a project sits on the 5-level framework
- **Journeys** — Real stories of upgrading projects (Pie: from 3 hours of frustrated debugging to all scenarios passing)
- **Scenario Patterns** — How to build external holdout test suites that agents can't game

## Quick Start

1. Read `research/dark-factory-research.md` for the full landscape
2. Read `assessments/harness-assessment.md` to see how 7 real projects were evaluated
3. Read `journeys/pie-journey.md` for the most detailed upgrade story
4. Copy templates from `templates/` into your project
5. Build external scenarios following `scenarios/README.md`

## The Core Loop

```
Interview → Spec → Fresh Session → Execute → External Scenarios →
Failures? → Feed back → Iterate → All Green → Ship
```
