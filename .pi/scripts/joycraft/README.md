# Joycraft Pi Scripts

Bash scripts that form the tool belt for Joycraft's autonomous Pi pipeline.

## Scripts

| Script | Purpose |
|--------|---------|
| `joycraft-spec-status` | Read `.joycraft-spec-queue.json` and print a formatted status table (glyphs: `[ ]` todo, `[~]` in-review, `[✓]` done) |
| `joycraft-mark-done` | Transition a spec's status in the queue: `joycraft-mark-done <id> --to <state>` where `<state>` is `todo`, `in-review`, or `done` (omitting `--to` defaults to `in-review`) |
| `joycraft-next-spec` | Find the next `todo` spec whose dependencies are satisfied (`in-review`/`done`), respecting dependency order |
| `joycraft-session-end` | Capture discoveries, run validation, and stage changes (the once-per-feature finisher) |
| `joycraft-implement-loop` | Isolated-mode driver: run a whole feature's queue headlessly, one fresh `pi -p` process per spec |

Status vocabulary is defined canonically in `docs/reference/spec-status-lifecycle.md` (`todo → in-review → done`).

## Usage

All scripts are designed to be called from the project root.

```bash
# Check status of all specs (3-glyph table)
.pi/scripts/joycraft/joycraft-spec-status

# Mark spec #3 in-review (spec-done), or graduate it to done (session-end)
.pi/scripts/joycraft/joycraft-mark-done 3 --to in-review
.pi/scripts/joycraft/joycraft-mark-done 3 --to done

# Get path of next spec to implement
.pi/scripts/joycraft/joycraft-next-spec docs/features/<slug>/specs

# Run the isolated-mode loop over a feature's queue (fresh process per spec)
.pi/scripts/joycraft/joycraft-implement-loop docs/features/<slug>/specs

# End a feature (validate + stage)
.pi/scripts/joycraft/joycraft-session-end add-pi-skills
```

`joycraft-implement-loop` reads the `pi` binary from `PI_BIN` (defaults to `pi`), so it can be tested with a stub and pointed at any Pi build. It is for Pi with a BYO API key or open-weight model — not a Claude/ChatGPT subscription OAuth (see the ToS note in the north star).

## Dependency

These scripts parse `.joycraft-spec-queue.json` — a JSON manifest generated by the `joycraft-decompose` skill. They use only POSIX-compatible `grep` and `sed` (no `jq` dependency).

## Pi Pipeline Flow

```
joycraft-implement-loop  (one fresh pi -p process per spec)
  next-spec → implement → spec-done (todo→in-review + commit) → repeat
                                      ↓
                          queue exhausted → session-end (validate, graduate
                                            in-review→done, push, PR) once
```
